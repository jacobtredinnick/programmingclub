Challenge - Tickle

Objective

Random movement based on mouse position

  • Easy:Change the word "tickle" to something else
  • Medium:Make sure the text never gets tickled out of the window
  • Hard:Make the text jump to a random location when it hits the edge of the window

    • Tools

    The starter program is a copy one of the Processing example programs.

    Easy Challenge hints

    Look for "tickle" and change it! Well it should be that easy but you will have to spot something else...

    Medium Challenge hints

    Here is the code that adjusts the text position in the draw() function: 
    
  // If the cursor is over the text, change the position
  if (abs(mouseX - x) < hr &&
      abs(mouseY - y) < vr) {
    x += random(-5, 5);
    y += random(-5, 5);
  }
    You need to add a check after this random adjustment to see if the new x or y value will push the text out of the window and set it back to the edge if so. For example this is how to stop the text going off the left edge : 
    
    // make sure the text does not get tickled off the screen
    if (x < hr) {
      x = hr;
    }
    The random movements in this example make it very hard to test your changes. Consider temporarily editing the other code to make it somehow easier to test. Perhaps we can change the random adjustment to make it more likely to hit the edge in normal use?

    Hard Challenge hints

    This is not really much harder than the medium challenge. Start with the medium solution (link below) and set a random x and y value when it hits the edge.
