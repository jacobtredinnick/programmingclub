Challenge - Rotating Cube

Objective

Draw some 3D graphics

Task

  • Easy: Adjust the starter animation to make the cube rotate in a different direction
  • Medium: Make the cube rotate in two directions and vary the speed of the rotation
  • Hard: Make the cube both rotate and move

    • Tools

    The starter program has a 3D animation where the cube rotates about the X axis. In 3D graphics there is an X and a Y axis (which we have used before) and there is also a Z axis (for the 3rd dimension). We don't have to use the Z axis in this challenge but you can if you want.

    Easy Challenge hints

    The rotation of the cube in the starter program is done in the draw() function:

    
		  rotateX(rotation); // makes the box draw with a specific rotation

    How would you expect to change the function to rotate about the Y or Z axis?

    Medium Challenge hints

    You can call multiple rotate... functions to get a 3D rotation

    
		  rotateX(...
		  rotateY(...
		  rotateZ(...

    The speed of the rotation is currently constant:

    
		  final float ROTATION_PER_FRAME = .01; // change in rotation in each frame

    Try turning this into a variable that uses, say, mouseX

    Hard Challenge hints

    The position of the cube is set by the translate function:

    
          translate(400,400); // makes the box draw with centre here

    You will have to introduce a variable that is set every frame, like in previous challenges...

    For collision detection (bouncing off the screen edge) you can do something like we did in the "bouncing ball" challenges, but it will be a bit harder to get the collision exactly right in 3D...
