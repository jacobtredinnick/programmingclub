Challenge - Bouncing Ball

Objective

Make an animation bounce vertically or horizontally

Task

  • Easy: Complete the starter program to make the ball bounce off the top and bottom edges of the screen
  • Medium: Make the ball move horizontally and bounce off the left and right edges of the screen
  • Hard: Make the ball gradually speed up while bouncing off both sides

    • Tools

    The starter program has a ball animation where the ball starts in the center, bounces off the bottom and goes off screen at the top.

    Easy Challenge hints

    The ball already bounces off the bottom of the screen and we need to make it bounce off the top using a similar trick.

    The bounce at the bottom of the screen is done by reversing the velocityY value at the end of the draw() function: 

    
  // reverse the ball direction if it hits the bottom edge
  if (ballY >= SCREEN_HEIGHT - BALL_WIDTH/2) {
    velocityY = -velocityY;
  }
    This code is saying : if the y position is greater than or equal to (the '>=' operator)the screen height minus half the ball width, the bottom edge of the ball hits so we must reverse direction. Try adding a similar check for if the ball is at the top of the screen by using the '<=' operator to check the ball position at the top of the screen.

    Medium Challenge hints

    The vertical animation is done in the draw() function here: 

    
  ballY += velocityY;
    This changes the y position each time draw() is called so that we when we draw the ellipse it is at a different position. What will have to change to move horizontally?

    Hard Challenge hints

    Start with either the vertical or horizontal bounce. To make something go faster we must increase its velocity. Adding something each time through the draw() should do it - but what if the velocity is negative?
